# Markdown ユーティリティ設計

## M1: Markdown ファイル一覧の取得

指定されたディレクトリ配下を再帰的に走査し、拡張子が `.md` のファイル一覧を返す。
戻り値は入力ディレクトリからの相対パスで、昇順に整列する。

## M2: Markdownファイル群のリンク切れ検出関数

`findBrokenLinks(rootDir)` は、`rootDir` 配下の Markdown ファイルに含まれる相対リンクを検査し、存在しないリンク先を返す。

対象にするリンク:

- インラインリンク `[label](target)` のみを対象にする。
- `http://`、`https://`、`mailto:`、`#` で始まるリンクは対象外にする。
- 画像リンク `![alt](target)` は対象外にする。
- リンク先の `#fragment` はファイル存在確認では無視する。

戻り値:

- 配列で返す。
- 各要素は `{ file, target }` の形にする。
- `file` はリンクを書いている Markdown ファイルの、`rootDir` からの相対パス。
- `target` は Markdown に書かれていたリンク先文字列をそのまま入れる。
- 戻り値は `file`、`target` の順で昇順に整列する。

## M3: CLI 出力

検出結果を標準出力に表示する CLI を提供する。
